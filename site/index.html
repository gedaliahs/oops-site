Undo for your terminal

oops watches for destructive commands, silently backs up affected files, and restores them when you need them back.

$ curl -fsSL oops-cli.com/install.sh | bash

macOS and Linux · other install methods

~ $ rm -rf src/
▲ rm -r ~/project/src

~ $ oops
✓ Undid: rm -r ~/project/src
↩ restored ~/project/src

How it works

Three pieces. No configuration required.

1

Shell hook intercepts

A preexec hook pattern-matches your command. Non-destructive commands pass through with zero overhead.

2

Files get backed up

Destructive commands trigger a backup of affected files to ~/.oops/trash/ in about 10ms.

3

Type oops to undo

Files are restored to their original paths. Use oops 2 or oops 3 to go further back.

Not just rm. oops handles the full range of destructive terminal operations.

Get started

Install the binary and add the shell hook. Under a minute.

1. Install

# one-liner curl -fsSL oops-cli.com/install.sh | bash # or homebrew brew install gedaliahs/oops/oops

2. Add the shell hook

# zsh — add to ~/.zshrc eval "$(oops init zsh)" # bash — add to ~/.bashrc eval "$(oops init bash)" # fish — add to config.fish oops init fish | source
