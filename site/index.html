oops

Like Time Machine for your terminal. Watches for destructive commands and lets you undo them.

$ brew install oops

macOS & Linux · other methods

~ $ rm -rf src/
▲ rm -r ~/project/src
 
~ $ oops
✓ Undid: rm -r ~/project/src
↩ restored ~/project/src

How it works

A shell hook intercepts destructive commands, backs up affected files, then lets the command run normally.

01

Hook matches the command

A lightweight preexec hook checks if your command is destructive. If not, it passes through with zero overhead—no subprocess spawned.

02

Files get backed up

For destructive commands, oops copies the affected files to ~/.oops/trash/ in about 10ms. Then your command runs normally.

03

Type oops to undo

Your files are restored to their original paths. Use oops 2 for second-to-last, or oops log to browse history.

Details

Fast, safe, stays out of your way.

~10ms overhead

Non-destructive commands have zero overhead. Destructive ones take about 10ms to back up. Benchmarked.

Risk scoring

rm -rf ~/ is not the same as rm tmp.log. High-risk gets a warning. Low-risk stays silent.

Auto-cleanup

Entries older than 7 days are purged automatically. Max 5GB trash. Both configurable.

Concurrent-safe

File-locked JSONL journal. Multiple terminals, same machine, no corruption.

All major shells

zsh, bash, and fish. Uses each shell's native preexec mechanism.

Single binary

Written in Go. No runtime dependencies. Download and run.
