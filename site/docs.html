Documentation

Installation, setup, and command reference.

Installation

Homebrew (macOS and Linux):

brew install oops

Direct download:

curl -fsSL https://raw.githubusercontent.com/gedaliahs/oops/main/install.sh | bash

From source:

git clone https://github.com/gedaliahs/oops.git cd oops go build -o oops . sudo mv oops /usr/local/bin/

Shell setup

oops needs a shell hook to intercept commands before they run. Add one line to your shell config:

zsh — add to ~/.zshrc:

eval "$(oops init zsh)"

bash — add to ~/.bashrc:

eval "$(oops init bash)"

fish — add to ~/.config/fish/config.fish:

oops init fish | source

Restart your shell or source the config file to activate.

To verify everything is working:

oops doctor

Basic usage

Once the hook is active, oops runs automatically. Use your terminal normally. When a destructive command runs, oops silently backs up the affected files before the command executes.

To undo the last destructive action:

# undo the last action oops # undo the second-to-last action oops 2 # undo the third-to-last oops 3

Example:

$ rm important-file.txt $ oops ✓ Undid: rm important-file.txt ↩ restored important-file.txt

Command reference

oops

Undo the last destructive action. Pass a number to undo the Nth most recent.

oops log

Show the undo history. Use -n 50 to show more entries.

oops size

Show total disk usage of oops backups.

oops clean

Remove old backups. Use --all to remove everything, or --older-than 3 to remove entries older than 3 days.

oops config

View or change settings. Run without arguments to see all values.

# show all settings oops config # change retention to 14 days oops config retention_days 14 # disable high-risk warnings oops config risk_warning false

oops doctor

Run a health check. Verifies the oops directory, config, git availability, and shell hook status.

oops init <shell>

Print the shell hook for zsh, bash, or fish.

oops completions <shell>

Generate shell completion scripts.

Configuration

Settings are stored in ~/.oops/config.json. Available keys:

  • retention_days — Days to keep backups before auto-cleanup. Default: 7.
  • max_trash_bytes — Maximum trash size in bytes. Oldest entries are removed when exceeded. Default: 5368709120 (5 GB).
  • risk_warning — Show a one-line warning before high-risk commands. Default: true.

Risk scoring

oops scores each destructive command by risk level:

  • Low — Temp files, build output, caches. Backed up silently.
  • Medium — Normal files, mv overwrites, sed -i. Backed up silently.
  • High — Recursive deletes, home directory files, config files. Shows a one-line warning before proceeding.

Risk is determined by: command type, flags (-rf, -f), file path (home vs temp), and whether the flag is recursive.

Storage

Everything lives in ~/.oops/:

~/.oops/ config.json # settings journal.jsonl # append-only action log .last_cleanup # cleanup timestamp trash/ 20260315-143022-a1b2/ # one dir per action manifest.json # file metadata files/ # backed up files

Auto-cleanup runs lazily on user-facing commands (at most once per hour). Entries older than retention_days are removed. If total trash exceeds max_trash_bytes, the oldest entries are removed first.

To manually clean up:

# remove entries older than 3 days oops clean --older-than 3 # remove everything oops clean --all

How it works

oops has three parts:

1. Shell hook. A preexec function (zsh), DEBUG trap (bash), or fish_preexec event (fish) that pattern-matches incoming commands. If the command starts with rm, mv, sed, git reset, etc., it calls oops protect. Otherwise it does nothing—no subprocess, no overhead.

2. Protect. The oops protect subcommand parses the shell command, identifies which files will be affected, and copies them to ~/.oops/trash/. It writes a journal entry and a manifest, then exits. Your original command runs normally after.

3. Undo. The oops command reads the journal, finds the most recent entry, reads its manifest, and copies the backed-up files back to their original paths. For git operations, it applies the stash or recreates the branch.
