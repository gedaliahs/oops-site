Documentation

Installation, shell setup, commands, and configuration.

Installation

One-liner (macOS and Linux):

curl -fsSL oops-cli.com/install.sh | bash

Homebrew:

brew install gedaliahs/oops/oops

From source (requires Go):

git clone https://github.com/gedaliahs/oops.git cd oops go build -o oops . sudo mv oops /usr/local/bin/

Shell setup

oops requires a shell hook to intercept commands before they run. Add one line to your shell config:

zsh — add to ~/.zshrc:

eval "$(oops init zsh)"

bash — add to ~/.bashrc:

eval "$(oops init bash)"

fish — add to ~/.config/fish/config.fish:

oops init fish | source

Restart your shell or source the config file to activate. Run oops doctor to verify the installation.

Usage

Once the hook is active, oops works automatically. Use your terminal normally. When a destructive command runs, oops backs up the affected files before the command executes.

# undo the last action oops # undo the second-to-last action oops 2 # undo the third-to-last action oops 3

Command reference

oops

Undo the last destructive action. Pass a number to go further back in history.

oops log

Show the undo history. Use -n 50 to show more entries.

oops size

Show total disk usage of oops backups.

oops clean

Remove old backups. Use --all to remove everything, or --older-than 3 to remove entries older than 3 days.

oops config

View or change settings. Run without arguments to see all current values.

oops config # show all settings oops config retention_days 14 # keep backups 14 days oops config risk_warning false # disable high-risk warnings

oops doctor

Run a health check. Verifies the oops directory, config, git availability, and shell hook status.

oops init <shell>

Print the shell hook for zsh, bash, or fish.

Configuration

Settings are stored in ~/.oops/config.json. Available keys:

  • retention_days — Days to keep backups before auto-cleanup. Default: 7.
  • max_trash_bytes — Maximum trash size in bytes. Oldest entries removed when exceeded. Default: 5 GB.
  • risk_warning — Show a one-line warning before high-risk commands. Default: true.

Storage

All data lives in ~/.oops/:

~/.oops/ config.json # settings journal.jsonl # append-only action log trash/ 20260315-143022-a1b2/ manifest.json # file metadata files/ # backed up files

Auto-cleanup runs lazily on user-facing commands, at most once per hour. Entries older than retention_days are removed. If total trash size exceeds max_trash_bytes, the oldest entries are removed first.

How it works

Shell hook. A preexec function (zsh), DEBUG trap (bash), or fish_preexec event (fish) pattern-matches each command. Safe commands pass through instantly with no subprocess spawned.

Protect. Destructive commands trigger oops protect, which parses the command, identifies target files, copies them to ~/.oops/trash/, and writes a journal entry. Takes approximately 10ms.

Undo. oops reads the journal, finds the most recent backup, and copies files back to their original paths. For git operations, it applies the stash or recreates the deleted branch.
