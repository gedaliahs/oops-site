Documentation

Installation, setup, and command reference.

Installation

One-liner (macOS and Linux):

curl -fsSL oops-cli.com/install.sh | bash

Homebrew:

brew install gedaliahs/oops/oops

From source:

git clone https://github.com/gedaliahs/oops.git cd oops go build -o oops . sudo mv oops /usr/local/bin/

Shell setup

oops needs a shell hook to intercept commands. Add one line to your shell config:

zsh — add to ~/.zshrc:

eval "$(oops init zsh)"

bash — add to ~/.bashrc:

eval "$(oops init bash)"

fish — add to ~/.config/fish/config.fish:

oops init fish | source

Restart your shell or source the config to activate. Run oops doctor to verify.

Usage

Use your terminal normally. When a destructive command runs, oops backs up the affected files automatically. To undo:

# undo the last action oops # undo the second-to-last action oops 2 # undo the third oops 3

Commands

oops

Undo the last destructive action. Pass a number to undo the Nth most recent.

oops log

Show undo history. Use -n 50 to show more entries.

oops size

Show disk usage of backups.

oops clean

Remove old backups. --all removes everything. --older-than 3 removes entries older than 3 days.

oops config

View or change settings.

oops config # show all oops config retention_days 14 # keep backups 14 days oops config risk_warning false # disable warnings

oops doctor

Health check. Verifies directory, config, git, and shell hook.

oops init <shell>

Print the hook for zsh, bash, or fish.

Configuration

Stored in ~/.oops/config.json:

  • retention_days — days to keep backups. Default: 7
  • max_trash_bytes — max trash size. Default: 5 GB
  • risk_warning — warn before high-risk commands. Default: true

Storage

Backups live in ~/.oops/:

~/.oops/ config.json # settings journal.jsonl # action log trash/ 20260315-143022-a1b2/ manifest.json # metadata files/ # backed up files

Auto-cleanup runs lazily (at most once per hour). Old entries get purged. If trash exceeds the max, oldest go first.

How it works

Shell hook — a preexec function pattern-matches your command. Safe commands pass through with zero overhead (no subprocess). Destructive commands trigger oops protect.

Protect — parses the command, identifies target files, copies them to trash, writes a journal entry. Takes about 10ms. Then your command runs normally.

Undo — reads the journal, finds the backup, copies files back to their original paths. For git operations, applies the stash or recreates the branch.
